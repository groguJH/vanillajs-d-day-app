# D-Day

사용자가 입력한 목표 날짜까지 남은 시간을 실시간으로 보여주는 간단한 D-Day 카운트다운 프로젝트입니다.  
Vanilla JavaScript로 날짜 검증, 타이머 상태 관리, `localStorage` 기반 복원 기능을 직접 구현했습니다.

## 프로젝트 소개

해당 프로젝트는 특정 날짜까지 남은 일, 시간, 분, 초를 한눈에 확인할 수 있는 개인 프로젝트입니다.

- 연, 월, 일을 입력해 원하는 날짜를 설정할 수 있습니다.
- 카운트다운 시작 버튼을 누르면 남은 시간이 실시간으로 갱신됩니다.
- 타이머 초기화 버튼으로 입력값과 저장 데이터를 함께 초기화할 수 있습니다.
- 잘못된 날짜, 만료된 날짜, 빈 입력값에 대해 각각 다른 안내 메시지를 보여줍니다.
- 마지막으로 설정한 날짜를 `localStorage`에 저장해 새로고침 이후에도 상태를 복원합니다.
- Tailwind CSS 기반 스타일링으로 간단하지만 깔끔한 UI를 구성했습니다.

## 기획 의도

- JavaScript만으로 날짜 계산과 실시간 카운트다운 흐름을 구현할 수 있다.
- 입력값 검증과 예외 상황 처리에 대한 기본적인 이해를 보여줄 수 있다.
- `localStorage`를 활용해 브라우저에 상태를 저장하고 복원하는 흐름을 구현할 수 있다.

## 주요 기능

### 1. D-Day 카운트다운 시작

- 연, 월, 일을 입력한 뒤 버튼을 누르면 카운트다운이 시작됩니다.
- 남은 시간은 일, 시간, 분, 초 단위로 나누어 표시됩니다.
- 1초마다 화면이 갱신되도록 구현했습니다.

### 2. 날짜 유효성 검사

- 존재하지 않는 날짜를 입력하면 안내 메시지를 출력합니다.
- 이미 지난 날짜를 입력하면 만료 메시지를 보여줍니다.
- 입력값이 비어 있으면 카운트다운 대신 기본 안내 메시지를 표시합니다.

### 3. 새로고침 이후 상태 유지

- 마지막으로 설정한 날짜를 `saved-date` 키로 저장합니다.
- 페이지를 새로고침해도 저장된 날짜가 있으면 자동으로 카운트다운을 다시 시작합니다.
- 초기화 버튼을 누르면 저장된 날짜와 입력값이 함께 삭제됩니다.

## 구현 포인트

### 1. 카운트다운 로직과 DOM 제어 분리

카운트다운 계산, 메시지 표시, DOM 갱신을 한 파일 안에 모두 섞지 않고 `createCountdownController` 형태로 분리했습니다.  
이 방식으로 초기화, 시작, 리셋 동작을 하나의 컨트롤러 안에서 일관되게 관리할 수 있도록 구성했습니다.

### 2. 유효하지 않은 날짜 판별

사용자가 `2025-02-31` 같은 존재하지 않는 날짜를 입력할 수 있기 때문에, 단순 문자열 조합만으로는 검증이 부족했습니다.  
그래서 `Date` 객체를 생성한 뒤, 다시 연/월/일 값을 비교해 실제로 유효한 날짜인지 판별하도록 구현했습니다.

### 3. 타이머 중복 실행 방지

카운트다운을 여러 번 시작하면 `setInterval`이 중복 실행될 수 있기 때문에, 기존 인터벌을 먼저 정리한 뒤 새 인터벌을 등록하도록 구현했습니다.  
이 과정을 통해 버튼을 여러 번 눌러도 타이머가 비정상적으로 빨라지지 않도록 처리했습니다.

### 4. 저장된 날짜 자동 복원

페이지 진입 시 `init()` 단계에서 저장된 날짜를 먼저 확인하고, 값이 있으면 바로 카운트다운을 시작하도록 구성했습니다.  
단순한 시연용 프로젝트가 아니라, 실제 사용 흐름을 고려한 상태 복원 경험을 보여주고 싶었습니다.

## 배포 시 참고사항

- 별도의 환경변수 없이 정적 배포가 가능한 프로젝트입니다.
- `localStorage`를 사용하므로 브라우저 환경에서 동작합니다.
- Vite 기반 프로젝트라서 정적 호스팅 환경에 배포하기 적합합니다.

## 느낀 점

이 프로젝트를 통해 단순한 타이머처럼 보이는 기능도 날짜 검증, 인터벌 관리, 상태 복원까지 함께 고려해야 한다는 점을 배웠습니다.  
특히 기능을 여러 파일로 나누어 관리하면서, 작은 프로젝트일수록 구조를 미리 정리하는 습관이 중요하다는 점을 느꼈습니다.
